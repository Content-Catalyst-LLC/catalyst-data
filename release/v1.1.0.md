# Catalyst Data v1.1.0

## Canonical Data Contract and Validation Engine

Catalyst Data v1.1.0 replaces the unversioned demonstration export with the canonical `catalyst-data-record/1.0` contract.

### Delivered

- Versioned JSON Schema with strict unknown-field rejection.
- Stable semantic IDs for records, entities, indicators, periods, and sources.
- Required creation/update timestamps and producer metadata.
- Complete source provenance fields: URL, publisher, license, retrieval date, citation, checksum, and access notes.
- Structured confidence basis, reviewer notes, assumptions, limitations, uncertainty, and quality flags.
- Namespaced extension rules that preserve a strict core contract.
- Runtime Python validation using JSON Schema plus cross-field semantic checks.
- TypedDict mappings alongside the canonical schema.
- Legacy v1.0.x record conversion and CLI upgrade command.
- Browser demo output aligned with the same canonical record structure.
- Packaged schema resources and release checks for schema parity.

### Compatibility

The Python API and CLI still accept v1.0.x authoring payloads and browser exports through explicit conversion. Canonical records are never silently repaired: malformed canonical fields, derived-value mismatches, unknown properties, invalid timestamps, invalid checksums, and invalid extension keys are rejected.

### Boundary

The contract makes records traceable and machine-valid. It does not certify source truth, regulatory compliance, impact, or professional conclusions.
## Installer reliability repair

- Isolated release checks from stale local Python bytecode during in-place upgrades.
- Added explicit cache removal to the macOS installer before validation.
- Added a regression contract ensuring cache isolation is configured before package import.
