# Catalyst Data v1.12.0

## Accessibility, Offline Use, Performance, and Release Hardening

Catalyst Data v1.12.0 completes the v1 operational hardening cycle. It adds verified backup and restore workflows, checksum-bound offline queues, persistent performance and security evidence, release attestations, and a more accessible and resilient WordPress public embed.

### Included

- `catalyst-data-operational-hardening/1.0` and packaged JSON Schema.
- Migration 012 with append-only backup, restore, synchronization, benchmark, security, and attestation history.
- SQLite online backup, manifest verification, pre-restore safety copies, and migration-aware restore.
- Offline queues for records, connectors, saved queries, analyses, and handoffs.
- Repository performance benchmarks and security audit checks.
- File manifests and a lightweight SBOM for release attestation.
- Accessible public embed controls, semantic record lists, keyboard focus, reduced-motion handling, retry, and clearly labeled cached fallback.

### Upgrade behavior

Existing v1.11.0 records, analysis artifacts, connector histories, workspaces, API keys, queries, reviews, provenance, and access history are preserved. Migration 012 adds operational history without changing the canonical record contract. Populated migration 012 can be rolled back after its append-only triggers are removed.

### Release boundary

The release provides local-first operational safeguards and evidence. It does not claim formal accessibility certification, regulatory compliance, or institution-wide disaster-recovery coverage.
