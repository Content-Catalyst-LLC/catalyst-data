# Catalyst Data v1.6.0 — Review, Quality, and Revision Workflow

Catalyst Data v1.6.0 adds the human-governed review layer required to move records from draft evidence into attributable, reproducible publication decisions.

## Added

- Backward-compatible `catalyst-data-review-workflow/1.0` records.
- Draft, submitted, in-review, changes-requested, approved, rejected, superseded, and archived states.
- Reviewer assignments, comments, decisions, priorities, and timestamps.
- Six-dimensional quality assessments with deterministic overall scoring and written bases.
- Blocked, internal, and external publication gates.
- Immutable approval snapshots bound to the exact approved payload checksum.
- Semantic revision diffs and current review-queue views.
- Review-aware JSON and CSV import/export.
- `reviews`, `review-history`, `review-assign`, `review-submit`, `review-start`, `review-decide`, `review-comment`, `quality-assess`, and `revisions` commands.

## Repository and migration

Migration 006 creates review cases, assignments, comments, decisions, quality assessments, approval snapshots, revision diffs, and current review views. Existing v1.5.0 records are preserved and backfilled through the canonical repository writer. The migration supports populated rollback and remigration.

## Compatibility

The canonical record remains `catalyst-data-record/1.0`. Previously published records remain valid; current authoring and repository workflows add and validate the review-workflow contract automatically.
