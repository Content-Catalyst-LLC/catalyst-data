# Catalyst Data v1.5.0 — Questions, Instruments, Datasets, and Observations

Catalyst Data v1.5.0 completes the first full question-to-measurement lineage path. It preserves the v1.4.0 indicator, methodology, unit, evidence, and provenance contracts while adding versioned collection systems and governed observations.

## Added

- `catalyst-data-observation-lineage/1.0` as a backward-compatible canonical-record property.
- Research, monitoring, evaluation, and decision questions.
- Versioned survey, sensor, form, API, administrative, model, manual, and other instruments.
- Versioned datasets with fields, licenses, access classifications, and checksums.
- Observation batches linking a dataset version to an instrument version.
- Baseline, current, supporting, and derived observations.
- Missing, censored, outlier, imputed, rejected, and valid observation states.
- Observation dimensions and raw-payload preservation.
- Explicit observation-to-measurement transformations.
- Append-only lineage events.
- Current instrument and dataset registries and a measurement-lineage summary view.

## Repository and migration

Migration 005 creates the persistent lineage tables and can be reversed safely after data has been populated. Existing v1.4.0 records are preserved and backfilled through the same repository writer used for new records.

## CLI

Added `questions`, `instruments`, `datasets`, `observations`, and `lineage` commands.

## Import and export

JSON imports accept complete observation-lineage objects. CSV imports and exports support `observation_lineage_json` and lineage count fields. Canonical JSON exports preserve the full lineage contract.

## Compatibility

The canonical record stays at `catalyst-data-record/1.0`. The new lineage contract is optional for previously published records and normalized automatically for current authoring and repository workflows.
