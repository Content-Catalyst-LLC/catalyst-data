# Catalyst Data v1.2.0 — Persistent Repository, Migrations, and Import Pipeline

Catalyst Data v1.2.0 converts the canonical record engine into a persistent, local-first SQLite application.

## Added

- Ordered and reversible SQL migrations with a migration ledger.
- Repository and application service layers.
- Persistent canonical records synchronized with normalized relational tables.
- Idempotent record upserts using stable record IDs and payload checksums.
- JSON and CSV import pipelines with dry runs, row-level errors, atomic rollback, and partial-import mode.
- Import run, imported-record, and import-error ledgers.
- JSON and CSV repository exports.
- Repository health, integrity, schema-version, statistics, inspection, and review commands.
- CLI commands: `init`, `migrate`, `rollback`, `status`, `import`, `export`, `inspect`, and `review`.
- Packaged migration SQL for installed Python distributions.

## Boundaries

This release provides local persistence and governed ingestion. Immutable revision history, multi-source provenance, institutional permissions, and remote APIs remain later roadmap work.
