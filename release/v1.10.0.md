# Catalyst Data v1.10.0

## Connectors, Refresh, and Data Operations

Catalyst Data v1.10.0 adds the governed operational layer needed to acquire, refresh, reconcile, and recover external evidence without weakening the repository's provenance, review, or workspace boundaries.

### Included

- `catalyst-data-connector-operations/1.0` and its packaged JSON Schema.
- Migration 010 with connector registry, versions, activations, schedules, state, runs, logs, immutable payload snapshots, row actions, source-key state, quarantine, dead letters, reconciliation reports, and operational alerts.
- JSON and CSV connectors for HTTP and local files.
- Explicit environment-variable credential references without stored secrets.
- Mapping and deterministic transformation profiles.
- Stable source keys, SHA-256 change detection, idempotent refresh, and workspace assignment.
- Manual, scheduled, retry, replay, and quarantine-recovery run types.
- Source freshness, licensing, schema-drift, record-drift, rate-limit, and health checks.
- Connector CLI and protected API endpoints.
- A free/open example connector and data feed.

### Upgrade behavior

Existing v1.9.0 records, workspaces, API keys, reviews, queries, provenance, and access history are preserved. Migration 010 adds connector structures without changing `catalyst-data-record/1.0`. The migration is reversible after populated connector operations; rolling it back removes connector operational history but leaves the canonical repository and migrations 001–009 intact.

### Release boundary

Connectors automate governed acquisition, not human approval. Imported records remain subject to quality review, workspace permissions, retention rules, and publication gates.
