# Catalyst Data v1.9.0 — Institutional Workspaces and Access Governance

Catalyst Data v1.9.0 adds multi-institution workspaces and role-governed record access while preserving the local-first repository, canonical record portability, public API, review workflow, provenance, and reproducible query system.

## Added

- Migration 009 for institutions, workspaces, projects, principals, memberships, retention policies, record access governance, API-key workspace bindings, access events, and transfer history.
- `catalyst-data-access-governance/1.0` schema.
- Seven workspace roles with explicit permissions.
- Record ownership, stewardship, custodianship, visibility, classification, retention, disposition, and legal-hold controls.
- Append-only access governance and workspace transfer events.
- Workspace-scoped API reads and dual enforcement of API scopes and workspace permissions.
- API keys bound to a principal and workspace.
- Public API publication gating that also requires public workspace visibility and classification.
- Auditable workspace export manifests.
- Institutional workspace and access-governance CLI commands.

## Upgrade behavior

Migration 009 creates a default Sustainable Catalyst institution, default workspace, system principal, and default retention policy. Existing records are assigned to the default workspace. Records that were already externally approved are backfilled as public/public; all others are private/internal.

Existing API keys are bound to the default workspace and system principal. New records receive a default workspace assignment automatically.

## Validation

The release suite covers:

- Fresh installation and migration 009 rollback/reapplication
- Populated v1.8.0 repository backfill
- Workspace isolation and role enforcement
- Public visibility and classification gating
- Legal holds and retention disposition checks
- Immutable access and transfer history
- API-key workspace binding
- Protected workspace HTTP endpoints
- Python package, schema, WordPress package, and macOS installer upgrade paths
