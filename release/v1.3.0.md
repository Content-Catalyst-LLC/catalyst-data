# Catalyst Data v1.3.0 — Sources, Provenance, and Evidence Chain

Catalyst Data v1.3.0 turns source provenance from a single record field into a versioned, queryable, append-only evidence system.

## Added

- Backward-compatible `catalyst-data-evidence-chain/1.0` contract.
- Multiple sources per measurement with primary, supporting, conflicting, derived, and contextual roles.
- Source locators, supported claims/fields, notes, and source-to-source relationships.
- Source version history and content snapshots.
- Immutable record revisions and append-only provenance events.
- Evidence transformations, derived gap detection, and completeness scoring.
- Current evidence-gap queue and evidence-chain summary views.
- `sources`, `provenance`, and `evidence` CLI commands.
- Evidence-aware JSON and CSV import/export.
- Repository API for manual source snapshots.

## Migration 003

Migration `003_sources_provenance_evidence` creates:

- `source_versions`
- `source_snapshots`
- `measurement_sources`
- `source_relationships`
- `record_revisions`
- `provenance_events`
- `evidence_gaps`
- `evidence_chain_summary`
- `open_evidence_gaps`

Immutability triggers reject updates or deletes against source versions, snapshots, record revisions, and provenance events. The data-bearing down migration safely dismantles append-only event chains before removing these additions, preserving the v1.2.0 repository layer. Existing v1.2.0 records are replayed through the canonical repository writer after migration so source versions, evidence links, revisions, events, gaps, and completeness metadata are backfilled consistently.

## Compatibility

The canonical measurement contract remains `catalyst-data-record/1.0`. `evidence_chain` is an optional backward-compatible property. The v1.3.0 record builder automatically enriches older canonical records with a primary-source evidence chain before repository storage.

The existing top-level `source` remains the compatibility primary source and must match the primary source embedded in `evidence_chain`.

## Review boundary

Evidence completeness is a transparent metadata score, not a truth score or certification. Catalyst Data records what evidence exists, what is missing or conflicting, and how records changed. Human reviewers remain responsible for interpretation and approval.
