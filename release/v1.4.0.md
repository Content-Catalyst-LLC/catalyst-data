# Catalyst Data v1.4.0 — Indicator Registry, Units, and Methodology Governance

Catalyst Data v1.4.0 adds a governed indicator layer above the persistent measurement and evidence repository introduced in v1.2.0 and v1.3.0.

## Delivered

- Backward-compatible `catalyst-data-indicator-governance/1.0` metadata on canonical records.
- Namespaced indicator codes, domains, custodians, aliases, definitions, lifecycle status, frequency, aggregation, and disaggregation dimensions.
- Governed unit definitions with dimensions, canonical conversion bases, scale factors, and offsets.
- Versioned methodology definitions with formulas, references, approval metadata, and revision notes.
- Immutable indicator and methodology version tables.
- Framework mappings that preserve distinct definitions instead of collapsing them.
- Compatibility rules for indicator versions, required dimensions, and methodology equivalence.
- Deterministic comparability results: equivalent, convertible, limited, or incompatible.
- Unit conversion through the repository and CLI.
- Append-only governance events.
- Automatic v1.3.0 record enrichment and registry backfill during migration 004.

## CLI additions

- `catalyst-data indicators DATABASE`
- `catalyst-data methods DATABASE`
- `catalyst-data units DATABASE`
- `catalyst-data convert DATABASE VALUE FROM_UNIT TO_UNIT`
- `catalyst-data compare DATABASE LEFT_RECORD RIGHT_RECORD`
- `catalyst-data governance-events DATABASE INDICATOR_ID`

## Compatibility

The canonical record remains `catalyst-data-record/1.0`. `indicator_governance` is an optional backward-compatible property for previously published records and is automatically normalized by the v1.4.0 builder and repository migration path.

## Boundary

Governance metadata records definitions, approvals, and comparability constraints. It does not certify the scientific validity, regulatory acceptance, or institutional authority of an indicator or methodology.
