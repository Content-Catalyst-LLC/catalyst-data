# Catalyst Data v1.0.1 — Repository Integrity and Package Contract Repair

## Release purpose

This patch establishes a trustworthy repository and package baseline before Catalyst Data adds persistent storage, migrations, APIs, or external connectors.

## Included repairs

- Added `VERSION` as the canonical release version source.
- Added generated Python and WordPress version artifacts.
- Added `contracts/review_contract.json` as the canonical evidence-review contract.
- Synchronized review thresholds, status labels, missing-source behavior, trace paths, and zero-baseline behavior across Python, SQLite, and JavaScript.
- Separated evidence readiness (`review_status`) from measurement direction (`signal_status`).
- Changed zero-baseline percent change from a misleading `0` to `null` / indeterminate.
- Added bounded confidence and direction validation.
- Added formal Python package metadata and a CLI entry point.
- Added deterministic WordPress packaging from committed source.
- Added repository and distribution release-contract checks.
- Added cross-runtime parity tests and expanded CI validation.
- Added unique form IDs for multiple WordPress shortcode instances.

## Compatibility

The export contract adds `signal_status` and permits `values.percent_change` to be `null` when the baseline is zero. Existing non-zero sample records remain structurally compatible apart from the added field and corrected review-status terminology.

## Validation command

```bash
python3 scripts/build_release.py
python3 scripts/check_release.py
```
