# Catalyst Data v1.8.0 — Public API, Embeds, and Platform Handoffs

Catalyst Data v1.8.0 opens the governed repository to public-safe reads and protected institutional exchange without weakening local-first operation.

## Added

- Migration 008 for scoped API clients, append-only request auditing, persistent embed profiles, immutable handoff receipts, and the externally approved record view.
- Dependency-light HTTP API with pagination, health, capability discovery, public records, protected record writes, typed handoff receipt, CORS, and OpenAPI 3.1.
- SHA-256-only bearer token storage, scopes, revocation, and last-use tracking.
- `catalyst-data-handoff/1.0` for checksum-bound exchange with Sustainable Catalyst products.
- Public projection rules that prevent internal review, restricted dataset, raw observation, and source-access metadata from leaking.
- Persistent WordPress shortcode `[catalyst_data_embed]` while retaining `[catalyst_data_demo]`.
- API key, server, OpenAPI, handoff creation, validation, receipt, and inspection CLI workflows.

## Compatibility

The canonical record remains `catalyst-data-record/1.0`. Existing v1.7.0 SQLite repositories migrate in place to schema version 8. Platform Core and remote API operation remain optional.
